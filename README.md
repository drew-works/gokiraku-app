# 🐙 人手不足と店舗インフラの課題を解決する「ハイブリッド型」注文決済システム
実際のアルバイト先（お好み焼き・もんじゃ焼き店「ごきらく亭」）が抱える「深刻な人手不足」と「店舗Wi-Fiの脆弱さ」というリアルな現場課題を解決するために開発した、新しいWeb注文システムです。

## ✨ 本プロジェクトにおける最大のこだわり（仕様改善・UX）

### 1. 店舗のインフラ制限を逆手に取った「オフライン対応QRコード」設計
- **現場の課題：** 一般的なQRコード注文は、お客様のスマホがお店のネット（Wi-Fi）に繋がることが前提です。しかし、当店はWi-Fiが弱く、スマホが圏外のようになってしまう問題がありました。
- **技術的な工夫と仕様改善：** ネット通信を必須とせず、**お客様のスマホ画面上で注文を確定させ、その場で注文内容が含まれた「QRコード」を生成する仕組み**にしました。店員側がそのQRをスキャンして伝票に書き写す形にすることで、店舗の電波状況に関係なく、確実に注文のデジタル化・時間短縮を実現しました。

### 2. アルバイトの経験から生まれた「業務効率化」と「注文ミス防止」
- **機会損失の削減：** これまでは混雑時に呼ばれても「まだ注文が決まっていない」と3分ほど待たされ、ホールの時間をロスすることがありました。システム化により、お客様が手元でじっくり選んでから店員を呼ぶ流れになり、大幅な時間短縮に繋がりました。
- **ステータス管理による注文忘れ防止：** ワンオペ（ホール1人）の時でもミスが起きないよう、店員側の画面で「提供済み」ボタンを押すと状態が変わる仕組みを導入。これにより、混雑時の注文忘れを仕組みで防げるようにしました。

### 3. デジタル化しても「お店の強み（接客・会話）」を消さない温かい設計
- **あえて完全非対面にしない選択：**
  完全にスマホだけで注文・決済が完結するシステムは、人手不足を解消する一方で、お店が大切にしている「お客様とのアットホームな会話」を消してしまいます。
  本システムでは、あえて「店員がお客様のスマホのQRをスキャンする」という1ステップを挟むことで、**業務を圧倒的に効率化しつつも、お客様と店員が必ず顔を合わせ、ちょっとした会話や温かみが生まれる接客の「余白」**を残しました。

## 🛠️ 使用技術
- HTML5 / CSS3（レスポンシブ対応）
- JavaScript（注文データの保持、QRコード生成ロジックの制御）
- **AI-Driven Engineering：**
  「Wi-Fiが使えない環境でどうやってデジタル注文を実現するか」「接客の温かみをどう残すか」という要件定義から、それを実現するロジック（JavaScriptでのQR生成など）への落とし込みをAIと共創しました。
